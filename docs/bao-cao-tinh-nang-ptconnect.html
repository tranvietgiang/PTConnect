1. Tổng quan hệ thống

Mục tiêu

  • Quản lý lớp học thêm môn Sinh học theo khối 10, 11, 12.
  • Theo dõi hồ sơ học sinh, sĩ số, lớp học và phụ huynh liên kết.
  • Hỗ trợ điểm danh theo từng buổi học, có trạng thái có mặt, đi muộn, vắng.
  • Giao bài tập, nộp file, tải tài liệu, chấm điểm và nhận xét.
  • Cho phụ huynh xem thông tin, lịch sử điểm danh, điểm số và bài tập của con.

Công nghệ sử dụng

React 19 Vite 8 Tailwind CSS 4 Axios Lucide icons Sonner toast

Laravel 12 PHP 8.2 MySQL 8 JWT Queue worker Mail

Triển khai local bằng docker-compose.yml gồm MySQL, backend, frontend, queue-worker và phpMyAdmin.

4. Bảng tính năng đã hoàn thành

Nhóm tính năng Đã làm Cách demo nhanh
Đăng nhập và phiên làm việc Đăng nhập bằng email hoặc username, JWT access token, refresh token, ghi nhớ đăng nhập bằng cookie, logout, tự khôi phục phiên. Đăng nhập staff ở /dang-nhap, phụ huynh ở /phu-huynh/dang-nhap.
Phân quyền Route API có jwt.auth và role; frontend có ProtectedRoute, redirect theo role và sidebar chỉ hiện menu được phép. Thử dùng teacher vào trang admin hoặc assistant vào điểm số sẽ bị chặn quyền.
Quản lý lớp học Danh sách lớp, chi tiết lớp, tạo lớp, cập nhật lớp, ngày bắt đầu/kết thúc, tổng số buổi, sĩ số, năm học active. Admin vào /lop-hoc và /lop-hoc/them.
Quản lý học sinh Danh sách học sinh, tìm theo tên/mã, lọc theo lớp, xem chi tiết, tạo hồ sơ, upload ảnh, import danh sách bằng Excel/CSV. Admin vào /hoc-sinh, dùng file docs/student-import-demo.xlsx.
Quản lý buổi học Admin tạo/sửa/xóa buổi học theo lớp, ngày, số buổi; chống trùng buổi học trong cùng lớp/ngày; lọc theo khối/lớp/ngày. Admin vào /buoi-hoc, chọn Lesson 1-30 theo tổng buổi của lớp.
Điểm danh Admin/assistant lấy danh sách học sinh theo lớp/ngày/buổi, đánh dấu có mặt, đi muộn kèm số phút, vắng, lưu lịch sử. Assistant vào /diem-danh, đổi trạng thái vài học sinh rồi gửi.
Thông báo điểm danh Khi học sinh đi muộn hoặc vắng, backend tạo notification recipient, email log và queue email thông báo phụ huynh. Sau khi gửi điểm danh vắng/muộn, phụ huynh xem lịch sử ở /phu-huynh.
Bài tập Admin/teacher tạo bài tập, giao theo lớp hoặc theo khối, hạn nộp, file đính kèm, tải tài liệu. Teacher hoặc admin vào /bai-tap, tạo bài tập mới.
Nộp bài và chấm điểm Phụ huynh nộp/cập nhật file bài làm cho con; giáo viên/admin tải bài nộp, nhập điểm 0-10, nhận xét, lọc đã nộp/chưa nộp/đã chấm. Parent nộp file ở /bai-tap, teacher chấm tại cùng trang.
Điểm số và báo cáo Danh sách điểm lấy từ bài nộp đã chấm, lọc khối/lớp/tên học sinh; báo cáo theo lớp gồm giỏi, khá, cần hỗ trợ, trung bình. Teacher/admin vào /diem-so và /diem-so/bao-cao.
Cổng phụ huynh Phụ huynh xem hồ sơ con, lớp, số bài đã chấm, điểm trung bình, lịch sử điểm danh, điểm số, file bài nộp và nhận xét. Parent vào /phu-huynh, /phu-huynh/diem-so, /bai-tap.
Dữ liệu demo và DevOps Seeder tạo năm học, lớp 10-12, học sinh, phụ huynh, bài tập, bài nộp, điểm, điểm danh và thông báo demo; Docker Compose đủ service local. Chạy seed rồi demo theo các tài khoản ở trang 2.

Trang /thong-bao hiện có giao diện danh sách/tạo thông báo mẫu. Luồng thông báo thực tế đã có ở backend khi điểm danh học sinh vắng hoặc đi muộn.

5. Luồng demo đề xuất

Khởi động hệ thống

Chạy docker compose up --build. Mở frontend http://localhost:5173, backend API http://localhost:8000, phpMyAdmin http://localhost:8080.

Admin - quản lý dữ liệu nền

Đăng nhập admin, mở Tổng quan, Lớp học, Học sinh. Tạo lớp mới, import học sinh bằng file Excel/CSV, xem chi tiết lớp và danh sách học sinh.

Admin - quản lý buổi học

Vào /buoi-hoc, tạo một buổi học cho lớp 10A1, sửa tên buổi hoặc ghi chú, lọc lại theo khối/lớp/ngày.

Assistant - điểm danh

Đăng nhập assistant, vào /diem-danh, chọn lớp/ngày/buổi. Đánh dấu một học sinh đi muộn, một học sinh vắng, sau đó gửi điểm danh.

Teacher - giao bài và chấm bài

Đăng nhập teacher, vào /bai-tap, tạo bài tập có hạn nộp và file đính kèm. Lọc bài nộp theo lớp/trạng thái, tải file bài nộp, nhập điểm và nhận xét.

Parent - xem thông tin con

Đăng nhập phụ huynh, mở /phu-huynh để xem hồ sơ, điểm trung bình và lịch sử điểm danh. Kiểm tra trạng thái đi muộn/vắng vừa tạo.

Parent - nộp bài và xem điểm

Vào /bai-tap để tải tài liệu và nộp file. Sau khi teacher chấm, vào /phu-huynh/diem-so để xem điểm, nhận xét và tải lại file bài nộp.

Kết luận demo

Mở /diem-so/bao-cao để cho thấy dữ liệu được tổng hợp theo lớp: số bài đã chấm, nhóm điểm và điểm trung bình.

7. Điểm nổi bật kỹ thuật

Bảo mật và phân quyền

  • JWT access token cho API nội bộ.
  • Refresh token và remember cookie HTTP-only cho chức năng ghi nhớ đăng nhập.
  • Middleware backend chặn request không có token hoặc sai vai trò.
  • Parent chỉ đọc dữ liệu học sinh liên kết trong parent_profiles.
  • Teacher/assistant bị giới hạn theo lớp phân công trong class_user_assignments.

Dữ liệu và file

  • Import học sinh từ CSV/XLSX, map được header tiếng Việt hoặc tiếng Anh.
  • Upload ảnh học sinh và file bài tập/bài nộp.
  • Download file đính kèm và bài nộp có kiểm tra quyền.
  • Seeder sinh dữ liệu demo đủ lớp, học sinh, phụ huynh, bài tập và điểm danh.

Trải nghiệm frontend

  • React Router bảo vệ route theo vai trò.
  • Sidebar hiển thị đúng menu cho từng user.
  • Toast phản hồi thành công/thất bại, trạng thái loading và form validation.
  • Bảng dữ liệu có lọc theo lớp, khối, tên học sinh, trạng thái bài nộp.

Thông báo và queue

  • Điểm danh vắng/đi muộn tự tạo notification và recipient.
  • Ghi nhận email log theo phụ huynh, học sinh và bản ghi điểm danh.
  • Docker có queue worker để xử lý mail queue trong môi trường demo.
